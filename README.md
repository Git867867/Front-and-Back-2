# Практическая работа №18 - Анализ доступности

## Выполнил: Магомерзаев Амир Русланович
## Группа: ЭФБО-13-24

## Проанализированные сайты:
1. Wildberries.ru - оценка 96
2. Ozon.ru - оценка 100  
3. Yandex.ru - оценка 84
4. GitHub.com - оценка 97
5. Google.com - оценка 99

## Основные проблемы доступности:

### 1. Wildberries.ru (96)
- **Контрастность текста** - некоторые элементы имеют недостаточный контраст для пользователей с ослабленным зрением
- **Атрибуты ARIA** - не все интерактивные элементы имеют правильные ARIA-роли

### 2. Ozon.ru (100) 
- **Ручная проверка** - требуется дополнительная ручная проверка 10 элементов
- **Навигация** - некоторые элементы навигации могут быть улучшены для скринридеров

### 3. Yandex.ru (84)
- **ARIA атрибуты** - неправильное использование ARIA-ролей на некоторых элементах
- **Альтернативные тексты** - часть изображений не имеет описательных alt-атрибутов

### 4. GitHub.com (97)
- **Размер кликабельных областей** - некоторые кнопки имеют недостаточный размер для удобного использования на мобильных устройствах
- **Сообщения об ошибках** - не все динамические уведомления правильно связаны с формами

### 5. Google.com (99)
- **Совместимость ARIA** - использование ARIA-ролей на неподходящих элементах
- **Управление фокусом** - некоторые динамические элементы могут быть улучшены для навигации с клавиатуры

## Вывод:
Большинство крупных сайтов имеют высокие показатели доступности, но даже у лидеров рынка есть возможности для улучшения, особенно в области ARIA-атрибутов и мобильной доступности.
